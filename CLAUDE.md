## Architecture (short)

3 linked repos under the `gorban-dev` org:

| Repo | Purpose |
|------|---------|
| `gor-mobile` (this one) | Bash CLI, wizard, templates for `~/.claude/` (hooks, skills, agents, rules pointer) |
| `gor-mobile-rules-default` | git pack: `manifest.json` + `rules/*.md` + `examples/<layer>/*.kt` |
| `homebrew-gor-mobile` | tap with `Formula/gor-mobile.rb` тАФ regenerated by CI on tag push |

Distribution: `brew install gorban-dev/gor-mobile/gor-mobile` тЖТ tap тЖТ release tarball.

## Release (critical)

The version lives in **two places** and must match:
- `lib/constants.sh` тЖТ `GOR_MOBILE_VERSION="X.Y.Z"`
- git tag `vX.Y.Z`

Release procedure:
1. Bump `GOR_MOBILE_VERSION` in `lib/constants.sh`.
2. Update `CHANGELOG.md`.
3. Commit, `git tag vX.Y.Z`, `git push --follow-tags`.
4. `gh release create vX.Y.Z --generate-notes` (or via UI).
5. CI (`release.yml`) automatically downloads the tarball, computes sha256, pushes the formula to the tap.

**Do not bump the tag without bumping the constant тАФ the wizard will show the old version in `gor-mobile version`.**

## Templates (`templates/` тЖТ `~/.claude/`)

Files from `templates/commands/`, `templates/agents/`, `templates/session-start-hook.sh`
are copied by the wizard into `~/.claude/`. After editing:
- The user needs to run `gor-mobile repair` to pick up the changes.
- Mention in CHANGELOG that `repair` is required.
- **Do not rename files** without a migration in `uninstall.sh` (leftovers will stay in `~/.claude/`).

## Managed sections

Markers that MUST NOT be changed/renamed тАФ the wizard uses them to locate its blocks:
- `<!-- BEGIN gor-mobile managed section -->` / `<!-- END gor-mobile managed section -->` (for `CLAUDE.md`)
- `"_managed_by": "gor-mobile"` (for `settings.json` hook entries)

If the marker changes, `uninstall`/`repair` stop seeing old installations.

## Rules pack compatibility

In `gor-mobile-rules-default/manifest.json`:
- `version` тАФ SemVer of the pack.
- `compatible_with` тАФ range of CLI versions that understand this pack.

Breaking changes (removing a section, renaming keys in `sections`,
changing the shape of `core.md`) тЖТ **bump major** + update `compatible_with`.
The CLI checks compatibility on `rules update`.

## CLI commands and slash commands

When adding a new CLI command (`lib/commands/foo.sh`):
1. Register it in `bin/gor-mobile` (dispatcher).
2. Add it to `gor-mobile help` (in `lib/commands/help.sh` if present, otherwise inline).
3. Update the "Commands" section in `README.md`.
4. Add a bats test in `tests/`.

When adding a new slash command (`templates/commands/foo.md`):
1. Use the `Task(subagent_type=..., model="sonnet"|"opus")` pattern from overlays when delegating.
2. Update the command list in `README.md` and `templates/session-start-hook.sh`.
3. Handle it in `uninstall.sh` (so it gets removed).

## Naming

- Everywhere `gorban-dev/`, **never** `gorban/` (the old, wrong one). If you spot `gorban/` тАФ fix it.
- CLI command: `gor-mobile` (with hyphen).
- Bash variables: `GOR_MOBILE_*` (upper-snake).

## Tests

The bats test suite was removed pending a rewrite тАФ do not add new bats
tests for now. Verify changes by running `gor-mobile init --dry-run --yes
--skip-sanity` and, when appropriate, a full wizard run in a sandbox
`HOME=$(mktemp -d)`.
