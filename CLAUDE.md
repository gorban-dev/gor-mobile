# gor-mobile — instructions for Claude

Rules to follow when working on this repo.
For every change — run through the checklist at the bottom.

## Architecture (short)

3 linked repos under the `gorban-dev` org:

| Repo | Purpose |
|------|---------|
| `gor-mobile` (this one) | Node + TypeScript CLI, install wizard, templates for `~/.claude/` and `~/.codex/` (hooks, skills, agents, rules pointer) |
| `gor-mobile-rules-default` | git pack: `manifest.json` + `rules/*.md` + `examples/<layer>/*.kt` |
| `homebrew-gor-mobile` | tap with `Formula/gor-mobile.rb` — regenerated by CI on tag push |

Distribution: `brew install gorban-dev/gor-mobile/gor-mobile` → tap → release tarball.
Alternate: `npm install -g gor-mobile` / `npx gor-mobile init` (both carry the
same `bin/gor-mobile.mjs` entrypoint that the brew formula wraps).

CLI implementation:
- `src/cli.ts` — Commander dispatcher.
- `src/commands/*.ts` — one file per subcommand (`init`, `doctor`, `repair`, …).
- `src/helpers/*.ts` — filesystem / settings / rules-pack helpers.
- `src/ui/*.ts` — `@clack/prompts` wrappers for the wizard (banner, welcome,
  section-header, progress, spinner, outro, note).
- `tsup` bundles to `dist/cli.js` (ESM, node20 target). `dist/` is committed
  so `brew install` / `npm install` can skip rebuild; `npm run build`
  regenerates it.

## Targets (`claude` | `codex`)

`src/targets.ts` is the single abstraction for *which agent* a run installs
into. `TargetSpec` resolves every per-agent path/format; helpers and commands
take a `TargetSpec` instead of hardcoding `CLAUDE_*`. Since v0.3.0 a spec also
carries a `scope` (`"user" | "project"`). The two static `TARGETS` entries are
user-scoped; `projectClaudeSpec(root)` builds a project-scoped Claude spec
rooted at `<repo>/.claude` (hooks → `settings.local.json`, no
`instructionsFile`, no status line). `init`/`repair`/`doctor`/`uninstall`
resolve scope by command mode (machine vs project) — there is no `--target`
flag on them anymore; only `setup` takes `--target codex` (Codex is the sole
user-level agent target left).

Layout per scope:

| Concern | Claude — **project** (`<repo>/.claude/`) | Codex — user (`~/.codex/`, `$CODEX_HOME`) |
|---------|------------------------------------------|--------------------------------------------|
| Hooks | `settings.local.json` `hooks` (not committed) | `hooks.json` (same JSON schema) |
| Skills | `skills/gor-mobile-*/SKILL.md` | `skills/gor-mobile-*/SKILL.md` (cross-compatible) |
| Agents | `agents/*.md` (`templates/agents/`) | `agents/*.toml` (`templates/agents-codex/`) |
| Instructions | injected by SessionStart hook (no file) | `AGENTS.md` managed section (same markers) |
| Plugins | `settings.local.json` `enabledPlugins` (superpowers off) | — |
| Status line | — (user-level, `setup` only) | `config.toml` `[tui].status_line` (built-in items) |
| Marker | `<repo>/.gor-mobile.json` (platform, version) | — |

The shared hook scripts (`~/.gor-mobile/templates/`) are target-neutral;
`session-start-hook.sh` reads its skills folder from `GORM_SKILLS_DIR` when the
Codex hook command sets it (always-on), and otherwise (bare command = Claude
project mode) walks up from cwd to the `.gor-mobile.json` marker and injects
from `<repo>/.claude/skills`. `android init` is run bare; `init` moves the stock
skill into `<repo>/.claude/skills` (Claude) and `setup` leaves Codex's home
copy.

## Release (critical)

The version lives in **two places** and must match:
- `src/constants.ts` → `GOR_MOBILE_VERSION = "X.Y.Z"`
- `package.json` → `"version": "X.Y.Z"`
- git tag `vX.Y.Z`

Release procedure:
1. Bump `GOR_MOBILE_VERSION` in `src/constants.ts` and `version` in `package.json`.
2. `npm run build` to refresh `dist/`.
3. Update `CHANGELOG.md`.
4. Commit, `git tag vX.Y.Z`, `git push --follow-tags`.
5. `gh release create vX.Y.Z --generate-notes` (or via UI).
6. CI (`release.yml`) downloads the tarball, computes sha256, pushes the
   formula to the tap. The formula runs `npm install && npm run build` at
   install time inside the brew sandbox.

**Do not bump the tag without bumping both constants — the wizard will show
the old version in `gor-mobile version`.**

## Templates (`templates/` → `~/.claude/` and `~/.codex/`)

Files from `templates/agents/` (Claude `.md`), `templates/agents-codex/`
(Codex `.toml`), `templates/skills/`, `templates/overlays/`,
`templates/session-start-hook.sh`, `templates/user-prompt-submit-hook.sh`
are copied by the wizard into the chosen target's home (or
`~/.gor-mobile/templates/` for the shared hook scripts).
`templates/banner.txt` is read at runtime for the `init` banner.
The Codex agent TOMLs carry the reviewer prose verbatim in
`developer_instructions`; keep them in sync with `templates/agents/*.md`.
After editing any template:
- The user needs to run `gor-mobile repair` to pick up the changes.
- Mention in CHANGELOG that `repair` is required.
- **Do not rename files** without a migration in `src/commands/uninstall.ts`
  (leftovers will stay in `~/.claude/` / `~/.codex/`).

## Managed sections

Markers that MUST NOT be changed/renamed — the CLI uses them to locate its blocks:
- `<!-- BEGIN gor-mobile managed section -->` / `<!-- END gor-mobile managed section -->` (for `CLAUDE.md`)
- `"_managed_by": "gor-mobile"` (for `settings.json` hook entries and `mcp.json` servers)

If the marker changes, `uninstall`/`repair` stop seeing old installations
(including installations created by the previous bash CLI).

## Rules pack compatibility

In `gor-mobile-rules-default/manifest.json`:
- `version` — SemVer of the pack.
- `compatible_with` — range of CLI versions that understand this pack.

Breaking changes (removing a section, renaming keys in `sections`,
changing the shape of `core.md`) → **bump major** + update `compatible_with`.
The CLI checks compatibility on `rules update`.

## CLI commands and slash commands

When adding a new CLI command (`src/commands/foo.ts`):
1. Export a `cmdFoo` function that performs the action.
2. Wire it in `src/cli.ts` via Commander (`program.command("foo")…`).
3. Update the "Commands" section in `README.md`.

When adding a new slash command or agent template (`templates/agents/foo.md`):
1. Use the `Task(subagent_type=..., model="sonnet"|"opus")` pattern from
   overlays when delegating.
2. Update any references in `README.md` and `templates/session-start-hook.sh`.
3. Confirm `src/commands/uninstall.ts` will remove it (prefix match on
   `gor-mobile-*` handles most cases).

## Naming

- Everywhere `gorban-dev/`, **never** `gorban/` (the old, wrong one). If you spot `gorban/` — fix it.
- CLI command: `gor-mobile` (with hyphen).
- TS constants: `GOR_MOBILE_*` (upper-snake, exported from `src/constants.ts`).

## Tests

No automated tests for now. Verify changes by running:

    npm run build
    npx tsc --noEmit
    ./bin/gor-mobile setup --dry-run --yes
    ./bin/gor-mobile init --dry-run --yes

and, when appropriate, a full roundtrip in a sandbox (setup is machine-level,
init is per-repo — run init from inside a repo dir):

    bash -c '
      TMP="$(mktemp -d)"
      export HOME="$TMP" GOR_MOBILE_HOME="$TMP/.gor-mobile" XDG_CONFIG_HOME="$TMP/.config"
      ./bin/gor-mobile setup --yes --no-tui
      REPO="$TMP/app"; mkdir -p "$REPO" && cd "$REPO"
      "$OLDPWD/bin/gor-mobile" init --yes --no-tui --platform android
      "$OLDPWD/bin/gor-mobile" doctor
      "$OLDPWD/bin/gor-mobile" uninstall --project --yes
      "$OLDPWD/bin/gor-mobile" uninstall --machine --yes
    '

## Stylistic

- TypeScript: `strict`, `noUncheckedIndexedAccess`. `npx tsc --noEmit` must be clean.
- Do not add blank lines at the end of files.
- Do not add `Co-Authored-By: Claude` to commits.
- Comments only when the WHY is non-obvious. The code should speak for itself.
