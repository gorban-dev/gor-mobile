# gor-mobile — Android Dev CLI — Contributor Guide for Claude

Bash CLI that wires a superpowers-style Android workflow into Claude Code and offloads routine codegen to local LLMs (LM Studio) with an Opus fallback.

This file lists the conventions Claude must follow when modifying this repo.
Run through the checklist at the bottom before every commit.

## Architecture (short)

3 linked repos under the `gorban-dev` org:

| Repo | Purpose |
|------|---------|
| `gor-mobile` (this one) | Bash CLI, wizard, LLM dispatcher, templates for `~/.claude/` |
| `gor-mobile-rules-default` | git pack: `manifest.json` + `rules/*.md` + `examples/<layer>/*.kt` |
| `homebrew-gor-mobile` | tap with `Formula/gor-mobile.rb` — regenerated by CI on tag push |

Distribution: `brew install gorban-dev/gor-mobile/gor-mobile` → tap → release tarball.

## Release (critical)

The version lives in **two places** and must match:
- `lib/constants.sh` → `GOR_MOBILE_VERSION="X.Y.Z"`
- git tag `vX.Y.Z`

Release procedure:
1. Bump `GOR_MOBILE_VERSION` in `lib/constants.sh`.
2. Update `CHANGELOG.md`.
3. Commit, `git tag vX.Y.Z`, `git push --follow-tags`.
4. `gh release create vX.Y.Z --generate-notes` (or via UI).
5. CI (`release.yml`) automatically downloads the tarball, computes sha256, pushes the formula to the tap.

**Do not bump the tag without bumping the constant — the wizard will show the old version in `gor-mobile version`.**

## Templates (`templates/` → `~/.claude/`)

Files from `templates/commands/`, `templates/agents/`, `templates/session-start-hook.sh`
are copied by the wizard into `~/.claude/`. After editing:
- The user needs to run `gor-mobile repair` to pick up the changes.
- Mention in CHANGELOG that `repair` is required.
- **Do not rename files** without a migration in `uninstall.sh` (leftovers will stay in `~/.claude/`).

## Managed sections

Markers that MUST NOT be changed/renamed — the wizard uses them to locate its blocks:
- `<!-- BEGIN gor-mobile managed section -->` / `<!-- END gor-mobile managed section -->` (for `CLAUDE.md`)
- `"_managed_by": "gor-mobile"` (for `settings.json` hook entries)

If the marker changes, `uninstall`/`repair` stop seeing old installations.

## Rules pack compatibility

In `gor-mobile-rules-default/manifest.json`:
- `version` — SemVer of the pack.
- `compatible_with` — range of CLI versions that understand this pack.

Breaking changes (removing a section, renaming keys in `sections`,
changing the shape of `core.md`) → **bump major** + update `compatible_with`.
The CLI checks compatibility on `rules update`.

## CLI commands and slash commands

When adding a new CLI command (`lib/commands/foo.sh`):
1. Register it in `bin/gor-mobile` (dispatcher).
2. Add it to `gor-mobile help` (in `lib/commands/help.sh` if present, otherwise inline).
3. Update the "Commands" section in `README.md`.
4. Add a bats test in `tests/`.

When adding a new slash command (`templates/commands/foo.md`):
1. Use strong wording about `gor-mobile llm <role>` if delegating to an LLM.
2. Update the command list in `README.md` and `templates/session-start-hook.sh`.
3. Handle it in `uninstall.sh` (so it gets removed).

## LLM routing

If you change routing in `lib/commands/llm.sh`, sync:
- The routing table in `README.md`.
- `templates/session-start-hook.sh` if behavior changes meaningfully.
- Tests in `tests/llm_test.bats`.

## Naming

- Everywhere `gorban-dev/`, **never** `gorban/` (the old, wrong one). If you spot `gorban/` — fix it.
- CLI command: `gor-mobile` (with hyphen).
- Bash variables: `GOR_MOBILE_*` (upper-snake).

## Tests

`bats tests/` must be green **before every commit**.
Run: `brew install bats-core` (once), then `bats tests/`.

## Stylistic

- Bash: `set -euo pipefail` in every script, `shellcheck`-clean.
- Do not add blank lines at the end of files.
- Do not add `Co-Authored-By: Claude` to commits.
- Comments only when the WHY is non-obvious. The code should speak for itself.

## Pre-commit checklist

- [ ] `bats tests/` green
- [ ] If `templates/` changed — mentioned `gor-mobile repair` in changelog
- [ ] If routing changed — synced README + tests
- [ ] If releasing — version in `constants.sh` matches the tag
- [ ] No `gorban/` slipped in instead of `gorban-dev/`
- [ ] No trailing blank lines
